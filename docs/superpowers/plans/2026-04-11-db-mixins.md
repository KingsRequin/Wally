# Database Mixins Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Découper `bot/db/database.py` (2145 lignes, 1 classe monolithique) en 8 mixins thématiques dans `bot/db/mixins/`. Zéro changement d'interface pour les appelants — `Database` hérite de tous les mixins.

**Architecture:** Pattern mixin Python pur. `Database(CostMixin, EmotionMixin, TrustMixin, MemoryMixin, SocialMixin, ChatMixin, GalleryMixin, ActionMixin)`. Chaque mixin accède à `self._conn`, `self.fetch_all()`, `self.fetch_one()`, `self.execute()` via le MRO. `database.py` garde uniquement : SCHEMA, `__init__`, `create`, `close`, `fetch_all`, `fetch_one`, `execute`, `upsert/get/delete/list_persistent_note`.

**Tech Stack:** Python, aiosqlite, pytest

## Préambule pour chaque mixin

Chaque fichier mixin démarre avec :

```python
from __future__ import annotations

import time
from typing import TYPE_CHECKING

import aiosqlite
from loguru import logger

if TYPE_CHECKING:
    pass  # ajuster si des types spécifiques sont nécessaires
```

La classe mixin déclare les attributs/méthodes utilisés du parent pour les type checkers :

```python
class XxxMixin:
    _conn: aiosqlite.Connection

    async def fetch_all(self, query: str, params=()) -> list: ...
    async def fetch_one(self, query: str, params=()) -> "aiosqlite.Row | None": ...
    async def execute(self, query: str, params=()): ...
```
